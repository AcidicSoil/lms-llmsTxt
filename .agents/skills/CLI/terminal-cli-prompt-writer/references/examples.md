# Examples

## Prompt Polisher

Input: "make this idea clearer: cli should look modern and rich"

Output:
```markdown
Build a polished command-line experience with structured, readable terminal output. Use clear section headers, aligned columns, helpful spacing, and restrained ANSI styling so results are easy to scan in a standard terminal.
```

## Design Brief Generator

Input: "need a brief for a modern cli look"

Output:
```markdown
Create a modern terminal-native CLI experience that feels professional, readable, and efficient. The interface should remain a normal command-line tool while using formatting to improve scanability: clear sections, aligned tables, subtle emphasis, status indicators, and optional box-drawing borders where they add clarity.
```

## CLI UX Prompt Writer

Input: "turn terminal-native cli design into a user prompt"

Output:
```markdown
Build a modern terminal-native CLI with structured, ANSI-styled output.

The CLI should feel polished, scannable, and professional while remaining a standard command-line tool, not a full-screen TUI. Use color and formatting to improve readability: clear section headers, aligned columns, color-coded commands/options/arguments, readable spacing, and subtle box-drawing borders where useful.

The desired style is language-agnostic. Do not assume Python, Typer, Rich, or any specific framework. Focus on terminal UX and output design rather than implementation technology.
```

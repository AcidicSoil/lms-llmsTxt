# CLI Terminal Skill Bundle

This bundle combines six reusable skill candidates derived from the CLI terminal design conversations.

## Included Skills

| Skill | Purpose |
|---|---|
| `cli-style-terminology` | Name and describe visual styles for command-line tools. |
| `terminal-ux-specification` | Turn CLI design preferences into reusable product/spec language. |
| `developer-design-vocabulary` | Translate screenshots or examples into clear engineering terminology. |
| `cli-prompt-builder` | Turn CLI design goals into clear implementation prompts. |
| `terminal-ux-spec-writer` | Write reusable specs for polished terminal-native CLI output. |
| `design-language-converter` | Convert visual or terminology notes into developer-ready prompts. |

## Core Language

Use this language-agnostic description when specifying the target CLI style:

> Build a modern terminal-native CLI with structured, ANSI-styled output. The CLI should feel polished, scannable, and professional while remaining a standard command-line tool, not a full-screen TUI. Use color and formatting to improve readability: clear section headers, aligned columns, color-coded commands/options/arguments, readable spacing, and subtle box-drawing borders where useful. The desired style is language-agnostic and should not assume Python, Typer, Rich, or any specific framework.

## Structure

Each skill folder contains a `SKILL.md` file with compliant frontmatter and concise activation instructions.

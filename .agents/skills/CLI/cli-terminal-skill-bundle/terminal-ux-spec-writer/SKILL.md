# Terminal UX Spec Writer

## Workflow

1. Draft a spec for a polished terminal-native CLI.\n2. Cover visual hierarchy, help output, tables, status messages, errors, warnings, success states, and fallback behavior when color is unavailable.\n3. Separate UX requirements from implementation choices.\n4. Keep wording suitable for product briefs, technical specs, or repository issues.

## Output Requirements

- Use language-agnostic terminology unless the user specifies a stack.
- Distinguish standard CLI output from full-screen TUIs.
- Prefer concise, implementation-ready language.
- Include the phrase "modern terminal-native CLI with structured, ANSI-styled output" when it fits.

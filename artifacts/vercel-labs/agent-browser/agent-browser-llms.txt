# Agent Browser

> The primary purpose of agent-browser is to provide AI agents with a fast, reliable, and deterministic interface for web automation. It enables agents to navigate, interact with, and extract data from web pages using semantic selectors and snapshot-based element references, which are optimized for LLM-driven workflows. The tool also supports advanced features like session isolation, streaming previews, and CDP connections, making it suitable for complex automation tasks including API testing, login flows, and real-time browser interactions.

**Remember:**
- Agent Browser Automation
- Semantic Locators (role, text, label, testid)
- Snapshot-based Element References (@e1, @e2)
- Rust CLI + Node.js Daemon Architecture
- Headless vs Headed Mode
- CDP (Chrome DevTools Protocol) Mode

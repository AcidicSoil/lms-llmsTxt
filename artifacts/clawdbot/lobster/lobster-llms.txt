# Lobster

> Lobster serves as a typed, deterministic workflow engine for Clawdbot, enabling agents to execute structured pipelines without constructing ad-hoc queries — thus conserving tokens and ensuring consistent behavior. It supports local-first execution, composable macros, and approval gates, making it ideal for automating tasks like PR monitoring, email triage, or code execution in controlled environments. The project aims to integrate seamlessly into Clawdbot’s plugin architecture while maintaining zero dependency on OAuth or new authentication mechanisms.

**Remember:**
- Typed pipelines (JSON-first)
- Local-first execution
- No OAuth/token ownership
- Composable macros
- Approval gates
- Determinism and resumability

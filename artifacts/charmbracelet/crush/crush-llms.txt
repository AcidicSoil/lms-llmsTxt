# Crush

> The primary purpose of Crush is to empower developers by integrating large language models into their terminal environment, enabling them to:  - Interact with codebases using natural language. - Switch between LLM providers or models mid-session. - Maintain persistent sessions with context-aware behavior. - Leverage LSPs and MCPs for enhanced context and extensibility. - Automate repetitive tasks via skills and custom tools. - Operate across multiple platforms without compromise.  Crush aims to reduce friction in AI-driven development, making it feel like a natural extension of the developer’s workflow — not an external tool.

**Remember:**
- Multi-Model Support
- Session-Based Context Preservation
- LSP Integration
- MCP Protocol Extensibility
- Terminal-First UX
- Agent Skills Framework

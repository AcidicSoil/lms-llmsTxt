# Oraclepack

> Oraclepack enables teams to create, validate, and execute self-documenting operational workflows embedded in Markdown files. It supports reproducible runs, real-time feedback via a TUI, state persistence, and integration with LLM evaluation pipelines. The tool aims to make complex multi-step operations (like audits, migrations, or LLM testing) manageable, auditable, and repeatable â€” turning documentation into executable runbooks.

**Remember:**
- Oracle Packs: Markdown files containing bash code blocks for operational workflows
- TUI (Text User Interface): Interactive shell-based interface for browsing, resuming, and managing workflow steps
- ROI (Return on Investment): A filtering mechanism to prioritize steps based on their value or complexity
- Artifact Gates: Validation mechanisms that check expected outputs from LLM executions
- State & Report Files: Persistent JSON files tracking execution status and metadata
- ChatGPT Project URLs: Global or project-scoped storage for managing prompts or contexts used in workflows
